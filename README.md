# Gemini 多账号自动生图工具

自动使用多个 Google 账号在 Gemini 网页端生成图片并下载的桌面工具，现已全面升级为 Web UI 控制台模式。

## ✨ 核心亮点

- **Web UI 任务队列** — 直接在网页输入提示词和目标张数，支持断点续传。
- **智能“抢单”机制** — 多窗口共享中央任务池，谁先跑完谁接下一单，算力绝不闲置。
- **生成间隔冷却** — 自定义每张图的生成间隔冷却时间，极大降低账号风控风险。
- **全自动防断连** — 额度用完自动换号，运行异常自动重启对话，保证连续作业。
- **无头不无脑** — 需要你手动介入登录，登录完它接管，真正做到安全第一。

## 🚀 启动方式

### 方式一：快捷方式启动（推荐 Windows 用户）
如果你在桌面上生成了 `启动跑图机器人.bat` 快捷方式，只需双击它，程序会自动在后台运行，并直接为你打开浏览器页面。

### 方式二：终端命令行启动
打开终端 (Terminal 或 cmd)，输入以下命令：
```cmd
cd 保存路径\gemini-auto-gen
python main.py
```

## 🛠 使用流程
1. 启动工具后，浏览器会自动打开 `http://127.0.0.1:5050`。
2. 配置**账号存放目录**和**图片输出目录**（只需配置一次）。
3. 设定**冷却时间**和**并行窗口数量**。
4. 在右侧 **“提示词任务队列”** 中添加你想要生成的提示词和目标张数。
5. 点击 **【1️⃣ 打开窗口准备登录】**。
6. 在弹出的 Chrome 窗口中，**手动完成 Google 账号的登录**（直到看到 Gemini 的输入框）。
7. 点击界面上的 **【2️⃣ 登录完成，开始跑图】**。
8. 脚本将全面接管浏览器，自动从任务队列里领任务、跑图、下载并保存到本地。

## 📁 目录结构
```
gemini-auto-gen/
├── main.py                    # 程序入口
├── web/                       # Web 服务端和前端 UI
├── core/
│   ├── browser_worker.py      # 浏览器工作核心逻辑
│   ├── gemini_automation.py   # Gemini 网页自动化控制 (自动打字、找图)
│   ├── task_queue.py          # 中央任务池控制器
│   ├── account_manager.py     # 多账号隔离配置管理器
│   └── image_downloader.py    # 图片处理与保存
├── prompts.json               # 任务数据存储文件（自动生成）
├── config.json                # 配置存储文件（自动生成）
└── logs/                      # 运行日志
```

## ⚠️ 注意事项
- **黑色的终端窗口（Terminal）在运行期间绝对不能关闭**。这是工具的服务端后台，一旦关闭网页就无法工作了。
- 本地生成的 Chrome 账号配置文件如果占用空间过大，可以定期清理不再使用的账号数据。
